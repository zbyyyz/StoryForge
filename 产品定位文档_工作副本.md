# StoryForge 产品定位文档

## 一句话定位

**创意到作品的转化器** —— 让有故事的人，不被笔力限制。

## 产品愿景

为"有想象力但缺笔力"的创作者提供一个低门槛、高上限的AI辅助创作工具。用户负责创意和判断（写什么、怎么走），AI负责执行力（怎么写得好看）。让用户觉得"这是我的作品，AI是我的笔力"。

## 目标用户

### 核心画像："有故事的人，缺笔力的人"

- 脑子里有情节、人物、画面感，但文笔/节奏/叙事技巧不足以把想象变成满意的成品
- 写出来的东西和脑中画面之间有落差，因此迟迟没有动笔或半途放弃
- 对成品质量有要求——能发表、能被人读、甚至能变现
- 愿意付出创意劳动（构思、设定、剧情编排），希望AI补齐"手艺"那一层
- 不排斥AI辅助，甚至认为这是理想的创作方式

### 用户来源（非酒馆圈）

- 番茄/起点的读者，偶尔想"我也能写"
- 小红书/B站看到AI写作但不知道怎么上手的人
- 用过ChatGPT写故事但觉得效果不可控的人
- LOFTER/AO3的读者，有同人/原创想法但没动笔的人

### 与酒馆用户的区别

| 酒馆核心用户 | StoryForge目标用户 |
|---|---|
| 技术能力强，享受DIY | 不想折腾，要省心 |
| 反感付费 | 愿意为好体验付费 |
| 把调参当乐趣 | 把创作本身当乐趣 |
| 已在圈内 | 从未进入过这个圈子 |

## 核心痛点

1. 现有工具（酒馆等）使用门槛过高，需要懂API配置、prompt工程、参数调试
2. 聊天式界面与"构思→成文"的创作流程不匹配
3. 对"骨架→扩写"过程缺乏精细控制
4. 长篇创作中世界观/角色状态难以管理，AI容易前后矛盾

## 核心差异化特性

### 1. 骨架与成文分离

用户的原始骨架（场景、剧情、对话、反应）作为独立层永久保留，AI扩写是覆盖在上面的另一层。不满意可以换风格、换参数重新生成，无需在聊天记录中翻找。

### 2. 创作资产复用与社区生态

世界观、角色模板、风格预设可跨作品复用和分享。高阶用户生产资产，新手消费资产，形成"高手建设生态，新手消费生态"的飞轮。

### 3. 渐进式复杂度（低门槛，高天花板）

- 新手：写一段话 → 点扩写 → 得到结果
- 进阶：选择风格预设、调整扩写策略
- 高级：世界观管理、角色状态追踪、多层风格叠加

## 借鉴与超越

### 从酒馆借鉴

- 世界观/Lorebook的结构化设定管理思路
- 角色卡的人设定义方式
- 社区资产分享的生态模式

### 超越酒馆之处

- 非聊天式界面，为"骨架→成文"工作流专门设计
- 零配置即可使用，无需API/参数知识
- 骨架与成文分离，支持非破坏性迭代
- 内置预设降低冷启动门槛

## 商业模式

### 核心收入：订阅制

- 免费层：基础扩写能力（有次数/字数限制），可使用社区资产
- 付费层（预估30-60元/月）：无限扩写、高级模型、风格学习、多作品管理

### 远期收入：资产市场抽成

- 用户可出售世界观包、角色模板、风格预设
- 平台抽成10-30%
- 前提：社区生态成熟后再开启

### 不做

- 阅读/发布平台（太重，偏离核心价值）

### 关键决策

- 作品版权完全归用户所有
- 主要成本为AI模型API调用费（活跃用户每月几元到几十元）

## 冷启动策略

不依赖社区内容冷启动，而是：

1. 平台内置高质量通用预设（现代都市、古风仙侠、西幻、校园等）
2. 用户进来选个类型，写几句话，立刻看到扩写效果
3. 资产市场是用户自然沉淀的结果，不是启动前提

## 核心交互流程："骨架→扩写"

### 三层创作流程

1. **故事构思（可选）**：用户写整体故事梗概 → AI建议章节划分和节奏 → 用户确认/调整
2. **章节骨架**：逐章写详细梗概（散文/叙述体，随意书写）
3. **扩写成文**：AI展开为完整文本

新手从第1层开始（AI引导分章），有经验的用户可直接从第2层开始。

### 输入：用户写骨架

- 形式：散文/叙述体的粗略梗概
- 内容：场景、动作、对话、人物反应、情绪等混合在一起，随意书写
- 不要求任何格式、分隔、结构化——想到哪写到哪，门槛最低

### 扩写：AI转化为成文

- 用户写完梗概后，选择一个"详细程度"（简洁 / 适中 / 细腻）
- 点击扩写，AI自动处理节奏、分段、描写密度
- 扩写单位由用户书写量自然决定，无需用户思考"该写多少"

### 原文保留策略

- 默认：AI自动判断——完整的对话句/精美表达倾向保留，概括性描述完全展开重写
- 进阶：用户可手动标记"保留原样"的句子（高天花板能力，非必须）

### 灵感建议（按需触发）

- 骨架编辑区提供"灵感"按钮，用户卡壳时点击
- AI根据当前骨架 + 角色/世界观设定，生成3-4个后续走向建议（梗概式，一两句话）
- 用户点击某个建议 → 追加到骨架末尾 → 可修改后再扩写
- 不主动弹出，不打断创作流；骨架为空时也可使用（冷启动）

### 迭代：局部微调

- 扩写完成后，用户可选中成文中的某一段进行局部调整
- "这里再展开一些" / "这里太啰嗦压缩一下"
- 无需重写骨架，直接在成文上微调

### 骨架与成文分离

- 用户的原始骨架永久保留为独立层
- 可随时切换回骨架视图，修改后重新扩写
- 换风格/换详细程度不会丢失原始创意

## 界面结构

### 平台形态

Web应用（浏览器打开即用，无需安装，跨平台）

### 主要界面

**1. 首页/工作台**
- 我的作品列表（卡片式：封面、标题、类型标签、进度、最后编辑时间）
- 快速开始入口（"新建作品"或"从模板开始"）
- 社区推荐的资产包

新建作品流程（引导式）：
选类型（都市/仙侠/西幻…） → 输入作品名和一句话简介 → 可选：写整体梗概让AI分章 → 进入创作

**2. 作品管理页**

左侧：章节目录
- 列表展示所有章节，状态标记（未开始/骨架已写/已扩写/已完成）
- "已完成"为提示性状态，不锁定，用户随时可改
- 可拖拽排序、新增、删除

右侧：作品信息面板
- 作品基本信息（标题、类型、简介）
- 快捷入口：角色库、世界观设定、风格预设（点击跳转独立设定管理页）
- 统计信息：总字数、各章字数、创作进度

**3. 创作工作区（核心界面）**

编辑器类型：富文本编辑器（支持标记保留原句等进阶功能）

骨架编辑态：
- 干净的富文本编辑框
- 侧边设定摘要面板（当前角色、场景背景速览）
- 底部：扩写按钮 + 详细程度选择（简洁/适中/细腻）

成文展示态：
- 展示AI扩写结果，用户可直接手动编辑文字（类似Word）
- 选中段落弹出操作菜单：展开更多/压缩/换一种写法
- 顶部切换按钮回到骨架视图

重新扩写保护机制：
- 重新扩写前提醒用户手动修改会被覆盖（A方案）
- 保留历史版本，随时可回溯（C方案）

对照模式（进阶）：
- 左右分栏，骨架与成文并排对照，检查是否遗漏情节点

**4. 设定管理界面**

角色卡结构：
- 基础（必填）：名字、一句话描述
- 进阶（按分类折叠，全部选填自由文本）：
  - 基本信息：性别、年龄/生日、身高、体重、语言
  - 身份背景：职业、家世、社会阶层、教育经历
  - 外在形象：外貌特征、发型发色、衣着风格、审美偏好、标志性物件
  - 性格内在：性格特征、价值观、习惯/癖好、恐惧/弱点、欲望/目标
  - 说话方式：语气、口癖、用词偏好、说话节奏
  - 行为模式：行事风格、社交方式、面对冲突的反应
  - 生活细节：住处、交通工具、日常作息、爱好
  - 人际关系：与其他角色的关系（可链接到其他角色卡）
  - 经历：关键人生事件、转折点、创伤/成长

世界观结构：
- 基础：类型标签选择（现代都市/古风仙侠/西幻/科幻/校园等）
- 进阶（按分类折叠，全部选填自由文本）：
  - 时空背景：时代、地理位置、气候、科技水平
  - 社会结构：政治体制、阶层划分、经济体系、主要势力/阵营
  - 规则体系：魔法/超能力规则、科技限制、物理法则特殊性
  - 文化风俗：宗教信仰、节日、禁忌、礼仪、语言特色
  - 历史事件：影响当前剧情的关键历史节点
  - 日常生活：货币、交通、通讯方式、饮食、娱乐
  - 冲突与张力：世界中存在的核心矛盾/暗线

风格预设管理（两者结合方案）：
- 第一层：命名预设作为起点（如"金庸武侠""余华冷叙事""日系轻小说""古风言情"）
- 第二层：在命名预设基础上微调各项参数
- 第三层：从零自定义
- 可调参数：叙事视角、文风倾向、描写偏好、节奏感、情感浓度、敏感内容尺度
- 调整后可保存为自定义预设

**5. 资产市场（远期）**

浏览页：
- 分类筛选：按类型（角色卡/世界观/风格预设/完整包）、按题材标签
- 资产卡片：名称、作者、简介、下载量/评分、免费或价格
- 支持单独分享（角色卡/世界观/风格预设各自独立）或打包分享

获取流程：
- 获取 → 存入"我的资产收藏" → 在作品设定管理页中选择"从收藏导入" → 应用到该作品

**6. 我的资产收藏**
- 统一管理从市场获取的所有资产
- 按类型分类展示
- 可从此处导入到任意作品

### 章节组织

- 默认按章组织，每章为一个独立的"骨架→扩写"单元
- 新手可通过"故事构思"功能让AI辅助分章
- 有经验的用户直接手动创建和管理章节

## MVP范围（Demo第一版）

### 目标

用最少的功能验证核心体验："写骨架→AI扩写→效果让人满意"

### 包含

- 创作工作区：骨架编辑 + 扩写 + 成文展示（单页面）
- 详细程度选择（简洁/适中/细腻）
- 成文可手动编辑
- 选中段落局部微调（展开更多/压缩/换一种写法）
- 风格预设选择（几个固定命名预设）
- 基础角色卡（名字+一句话描述）
- 基础世界观（类型标签选择）

### 不包含（后续迭代）

- 用户注册/登录（本地运行）
- 多作品/章节管理
- 故事构思→AI分章
- 角色卡进阶字段
- 风格预设微调/自定义
- 骨架与成文视图切换
- 历史版本
- 资产市场
- 对照模式
- 标记保留原句

### 技术方案

- 前端：React + Next.js
- 数据库：SQLite（本地文件）
- AI模型：Claude API（需申请API key）
- 部署：本地运行（localhost）

## 暂缓事项

### 沉浸式付费配色主题

想法：提供沉浸式配色主题作为付费功能，帮助用户在创作时更加沉浸。主题方向包括：古风、宫廷、江湖、西欧、都市、校园、未来。

要求：付费主题不能只是换颜色，必须有独特的视觉设计（纹理、图案、装饰元素等），让用户觉得物有所值，明显优于免费主题。

状态：当前原型效果不够精美，暂缓实现。等有条件（如请专业设计师）时再做。
