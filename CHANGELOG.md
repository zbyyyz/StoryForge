# StoryForge 设计迭代记录

本文档记录 StoryForge 产品设计与开发的关键决策和迭代历程。

---

## 2026-05-29 项目基线建立

### 当前状态

- 完成产品定位文档（目标用户、核心流程、界面结构、商业模式）
- 完成 10 个 HTML 静态原型（首页工作台多风格探索、新建作品、作品管理、创作工作区）
- 完成 Next.js Demo 第一版，已实现核心功能：
  - 骨架编辑 → 一键扩写（5 种风格 + 3 种详细程度）
  - 成文后局部微调（展开/压缩/换写法）
  - 基础角色管理 + 世界观类型选择
  - 骨架/成文视图切换
- 技术栈：Next.js 16 + React 19 + Tailwind CSS 4 + DeepSeek API

### 已确定的设计决策

1. 核心交互为"骨架→扩写"，非聊天式
2. 骨架与成文分离，支持非破坏性迭代
3. 渐进式复杂度：新手零门槛，高手有天花板
4. 订阅制商业模式（30-60元/月）
5. 不做阅读/发布平台
6. 沉浸式付费配色主题暂缓（等专业设计师）

### MVP 已实现 vs 未实现

已实现：创作工作区核心流程（骨架→扩写→微调）

未实现：多作品/章节管理、完整角色卡、世界观详细设定、风格微调/自定义、历史版本、资产市场、用户系统
