## 2026-06-01 新增设计决策：故事级预设参数

### 背景

用户需要对整体故事走向进行更精确的控制，现有文风级参数无法满足需求。

### 新增参数

| 参数项 | 类型 | 可选值 |
|--------|------|--------|
| 对话语言风格 | 单选 | 文雅、日常、古典、粗犷、卖萌、冷峻 |
| 剧情发展方向 | 单选 | HE（大团圆）、BE（悲剧）、OE（开放式） |
| 故事侧重点 | 单选 | 阴谋、感情拉扯、事业、成长、探案、群像 |
| 节奏偏好 | 单选 | 快速推剧情、注重细节描写、均衡 |
| 性向 | 单选 | BG（男女）、BL（男男）、GL（女女）、无CP |
| 世界基调 | 单选 | 主角万能、爽文、现实向、虐文、轻松 |

### 参数分类

- **文风级参数**：影响文字表达方式（叙事视角、文风倾向、描写偏好等）
- **故事级参数**（新增）：影响整体剧情走向
- **输出密度参数**：影响扩写篇幅（详细程度）

### UI位置

建议放在"角色/世界观"侧边栏，作为"故事设定"板块；或在新建作品流程中，作为作品级设定。

### 实现要求

1. 全部为可选参数，不填有默认值
2. 参数需传递给 `/api/expand` 接口
3. 在 system prompt 中注入对应指令

## 2026-05-29 项目基线建立

### 当前状态

- 完成产品定位文档（目标用户、核心流程、界面结构、商业模式）
- 完成 10 个 HTML 静态原型（首页工作台多风格探索、新建作品、作品管理、创作工作区）
- 完成 Next.js Demo 第一版，已实现核心功能：
  - 骨架编辑 → 一键扩写（5 种风格 + 3 种详细程度）
  - 成文后局部微调（展开/压缩/换写法）
  - 基础角色管理 + 世界观类型选择
  - 骨架/成文视图切换
- 技术栈：Next.js 16 + React 19 + Tailwind CSS 4 + DeepSeek API

### 已确定的设计决策

1. 核心交互为"骨架→扩写"，非聊天式
2. 骨架与成文分离，支持非破坏性迭代
3. 渐进式复杂度：新手零门槛，高手有天花板
4. 订阅制商业模式（30-60元/月）
5. 不做阅读/发布平台
6. 沉浸式付费配色主题暂缓（等专业设计师）

### MVP 已实现 vs 未实现

已实现：创作工作区核心流程（骨架→扩写→微调）

未实现：多作品/章节管理、完整角色卡、世界观详细设定、风格微调/自定义、历史版本、资产市场、用户系统

## 2026-05-29 新增设计决策：灵感建议功能

### 来源

借鉴酒馆（AI Tavern）扩写后给出 ABCD 选项的机制，但适配 StoryForge 的"用户掌控创意"定位。

### 设计方案

- 在骨架编辑区增加一个"灵感"按钮（按需触发，非自动弹出）
- 用户点击后，AI 根据当前骨架内容 + 角色/世界观设定，生成 3-4 个后续走向建议（每个一两句话，梗概式语气）
- 用户点击某个建议 → 文字追加到骨架编辑区末尾 → 用户可修改后再扩写
- 不点就关掉，继续自己写

### 关键原则

1. 按需触发，不主动打断有灵感的用户
2. 建议追加到骨架（不直接生成成文），保持用户掌控
3. 骨架为空时也能使用（冷启动场景）
4. 建议的语气是梗概式的，与骨架风格一致
